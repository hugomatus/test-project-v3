# Conversation Context

This directory stores all architectural conversations and their context for BluePrintIQ.

## Structure

- `turns/` - Individual conversation turns with AI responses
- `checkpoints/` - Conversation state snapshots for restoration

## Conversation Turns

Each conversation turn includes:
- User input and architectural intent
- AI analysis and recommendations
- Specification changes generated
- Context and metadata

## Context Restoration

BluePrintIQ uses conversation history to:
- Maintain architectural context across sessions
- Provide relevant responses based on project evolution
- Enable seamless collaboration between team members

*Powered by BluePrintIQ Conversation Engine*
